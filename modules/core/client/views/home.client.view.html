FIRMS - Field Inspection Report Management System (For project Checkpoint only)


Description:

Our project idea was inspired from a coop experience at a structural engineering office, where the company needed an automated method of writing/storing field reports submitted by their field inspectors. The field inspector would go out to construction sites, inspect the quality of the work and ensure if the site is up to the engineerâ€™s standards and instructions.


Inspection reports were initially written using paper forms and pen. The goal of our project will be to build a client/server web application that will automate the form generation process, with auto-complete features, and form management. Field inspectors will be able to login into a special website (client side), create a report, and have as many fields and associated data auto-filled for a given construction project. Many of these fields have unique relationships (e.g. contractors and developers working on project -- relational data in SQL database), while others are to do with auto-generating location and weather/temperature info (API calls - Google Maps, etc.). The inspector can either mark the status of their report as Passed, Failed, or Needs Re-Inspection.

Frameworks used:

  • Bulma - Frontend Framework
  • MEAN.js - Fullstack Framework

External APIs used:

  • Google Maps API - Places, Geolocation, Geocoding, Javascript
  • Weather API

Deployment:

  • Docker - Ubuntu Edition
  • Google Cloud Run
